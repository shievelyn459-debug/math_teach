# E2E 测试场景文档

# MathLearningApp

---

## 测试场景概述

本文档描述了 MathLearningApp 的端到端测试场景，覆盖核心用户流程和关键功能。

---

## 测试场景列表

### 场景 1: 新用户完整旅程

**描述**: 新用户从注册到完成第一次题目导出的完整流程
**优先级**: P0 (必须)
**预计时间**: 2-3 分钟

#### 测试步骤
1. 启动应用
2. 验证欢迎页面显示
3. 点击注册按钮
4. 填写注册表单（姓名、邮箱、密码）
5. 提交注册
6. 验证自动登录并跳转到首页
7. 点击添加儿童按钮
8. 填写儿童信息
9. 保存儿童档案
10. 点击拍照按钮
11. 拍摄题目照片
12. 硉认照片
13. 等待识别完成（30秒内）
14. 验证题型识别结果
15. 选择难度
16. 点击生成相似题目
17. 等待生成完成
18. 查看知识点讲解
19. 点击导出 PDF
20. 下载 PDF

**验收标准**:
- 每个步骤顺利完成
- 没有错误提示
- 识别时间 < 30 秒
- PDF 成功下载

### 场景 6: 网络错误处理
**描述**: 模拟网络不可用时的错误处理
**优先级**: P1 (重要)
**预计时间**: 30 秒

#### 测试步骤
1. 登录
2. 关闭网络
3. 尝试上传题目
4. 验证错误提示显示
5. 验证重试按钮可用
6. 恢复网络
7. 点击重试
8. 验证操作成功继续
**验收标准**:
- 错误提示清晰
- 重试功能正常
- 网络恢复后操作继续

---

### 场景 7: 权限拒绝处理
**描述**: 模拟相机权限被拒绝时的处理
**优先级**: P1 (重要)
**预计时间**: 30 秒

#### 测试步骤
1. 撤销相机权限
2. 启动应用
3. 登录
4. 点击拍照按钮
5. 验证权限请求提示显示
6. 验证拒绝后的引导信息
**验收标准**:
- 权限请求提示清晰
- 引导信息正确
- 用户可以前往设置页面
---

### 场景 8: 表单验证
**描述**: 验证各种表单的输入验证
**优先级**: P1 (重要)
**预计时间**: 1 分钟

#### 测试步骤
1. 打开注册页面
2. 提交空表单
3. 验证必填字段错误提示
4. 输入无效邮箱格式
5. 验证邮箱格式错误
6. 输入过短密码
7. 验证密码强度错误
8. 输入不匹配的确认密码
9. 验证密码匹配错误
**验收标准**:
- 所有验证规则正确执行
- 错误提示清晰
- 用户可以修正输入

### 场景 9: 超时处理
**描述**: 验证长时间操作的超时处理
**优先级**: P2 (一般)
**预计时间**: 1 分钟

#### 测试步骤
1. 登录
2. 上传题目
3. 等待超过 30 秒
4. 验证超时错误提示
5. 验证重试选项可用
**验收标准**:
- 超时错误正确显示
- 重试选项可用

3. 验证离线提示显示
4. 上传题目
5. 验证操作被排队等待

7. 验证自动同步
**验收标准**:
- 离线提示正确显示
- 操作正确排队
- 网络恢复后自动同步
- 数据一致性保持
