## 🚀 快速开始（3步完成）

### 第1步：PM填写功能清单

使用模板填写功能清单：

```markdown
## 功能名称：用户登录

**操作步骤**：
1. 输入邮箱密码
2. 点击登录
3. 验证进入首页

**期望结果**：登录成功
**代码位置**：src/screens/LoginScreen.tsx
```

### 第2步：保存清单

保存到项目目录：
```
MathLearningApp/docs/e2e-requests/登录功能.md
```

### 第3步：通知QA

告诉Quinn：
```
"功能清单已放在 docs/e2e-requests/登录功能.md"
```

## 💬 常见问题

### Q1：PM不懂技术，代码位置怎么填？
**A**: 可以留空，QA会自己分析代码找到相关文件

### Q2：功能清单需要多详细？
**A**:
- 最少：功能名 + 操作步骤 + 期望结果
- 推荐：参考标准版模板
- 复杂功能：使用详细版模板

### Q3：可以同时提交多个功能吗？
**A**: 可以，建议按优先级提交，QA会依次处理

### Q4：填错了怎么办？
**A**: 直接修改文件，告诉QA"xxx功能清单已更新"

### Q5：QA需要多长时间？
**A**:
- 简单功能：5分钟内完成
- 中等功能：15分钟内完成
- 复杂功能：30分钟内完成
