# E2E测试功能清单模板

> **使用说明**: PM或业务分析师填写此清单，QA（Quinn）将根据清单生成并执行E2E测试

---

## 📋 模板选择指南

| 功能复杂度 | 推荐模板 | 填写时间 |
|-----------|----------|----------|
| 简单功能（<3步操作） | 简单版模板 | 2分钟 |
| 中等功能（3-5步操作） | 标准版模板 | 5分钟 |
| 复杂功能（>5步/多分支） | 详细版模板 | 10分钟 |

---

## 🟢 模板一：简单版（推荐新手使用）

**适用场景**: 登录、退出、简单查询等单一路径功能

```markdown
### 功能名称：[填写功能名称]

**功能描述**：[一句话描述这个功能做什么]

**用户角色**：[谁在使用？例如：家长/孩子/管理员]

**操作步骤**：
1. [第一步操作]
2. [第二步操作]
3. [第三步操作]

**期望结果**：
- [成功后应该看到/得到什么]

**相关代码位置**：src/xxx/xxx.tsx（可选，不知道可不填）
```

**填写示例**：
```markdown
### 功能名称：用户登录

**功能描述**：家长使用邮箱和密码登录App

**用户角色**：家长

**操作步骤**：
1. 打开App进入登录页
2. 输入邮箱：test@example.com
3. 输入密码：password123
4. 点击"登录"按钮

**期望结果**：
- 登录成功，进入首页
- 显示用户名和头像

**相关代码位置**：src/screens/LoginScreen.tsx
```

## 🟡 模板二：标准版（推荐日常使用）

**适用场景**: 大部分业务功能

```markdown
## 功能名称：[填写功能名称]

### 功能描述
[详细描述这个功能的业务价值和使用场景]

### 用户角色
- 主要使用者：[谁主要使用]
- 其他角色：[还有谁会用到]

### 前置条件
- [必须满足的条件1]
- [必须满足的条件2]

### 操作流程

#### 主要流程（正常场景）
1. **[步骤1名称]**
   - 操作：[具体操作]
   - 界面：[在哪个页面操作]
   - 预期：[这一步应该看到什么]

2. **[步骤2名称]**
   - 操作：[具体操作]
   - 界面：[在哪个页面操作]
   - 预期：[这一步应该看到什么]

#### 异常场景（需要测试的特殊情况）
- **[异常情况1]**: [应该怎么处理]
- **[异常情况2]**: [应该怎么处理]

### 验收标准
- ✅ [必须满足的标准1]
- ✅ [必须满足的标准2]
- ✅ [性能要求：例如响应时间<3秒]

### 相关代码
- 功能页面：src/xxx/xxx.tsx
- 服务层：src/services/xxx.ts（如果有）
```

**填写示例**：
```markdown
## 功能名称：拍照上传题目

### 功能描述
家长通过拍照上传孩子的数学作业题目，系统自动识别题目类型、答案并生成讲解

### 用户角色
- 主要使用者：家长
- 其他角色：无

### 前置条件
- 用户已登录
- 已选择当前学习的孩子
- 已授予相机权限

### 操作流程

#### 主要流程（正常场景）
1. **打开拍照页面**
   - 操作：点击首页"拍照"按钮
   - 界面：首页 → 相机页面
   - 预期：显示相机预览，底部有拍照按钮

2. **拍摄题目照片**
   - 操作：对准题目，点击拍照按钮
   - 界面：相机页面
   - 预期：拍照完成，显示确认界面

3. **确认使用照片**
   - 操作：点击"使用这张照片"
   - 界面：确认页面 → 识别进度页
   - 预期：显示识别进度动画

4. **查看识别结果**
   - 操作：等待识别完成（最多30秒）
   - 界面：识别进度页 → 结果页面
   - 预期：显示题目、答案、讲解

#### 异常场景
- **照片模糊**: 提示"照片不清晰，请重拍"
- **网络异常**: 保存到本地，提示"网络恢复后自动同步"
- **识别失败**: 提示"无法识别，请重试或手动输入"

### 验收标准
- ✅ 识别准确率 ≥ 90%
- ✅ 识别时间 < 30秒
- ✅ 离线时也能保存照片
- ✅ 识别结果自动保存到学习记录

### 相关代码
- 功能页面：src/screens/CameraScreen.tsx
- OCR服务：src/services/ocrService.ts
- 学习记录：src/services/api.ts (recordStudy方法)
```

## 🚀 提交流程

1. **填写清单** → 使用适合的模板
2. **保存文件** → 保存到项目 `docs/e2e-requests/` 目录
3. **通知QA** → 告诉Quinn清单位置
4. **等待测试** → QA生成测试并执行
5. **查看报告** → 获取测试结果

## 💡 填写技巧

### ✅ 好的描述
```
操作：点击"登录"按钮
预期：进入首页，显示用户名
```

### ❌ 不好的描述
```
操作：点按钮
预期：成功了
```

### ✅ 好的验收标准
```
- ✅ 3秒内完成登录
- ✅ 错误提示显示在输入框下方
- ✅ 密码输入不可见
```

### ❌ 不好的验收标准
```
- ✅ 用户体验好
- ✅ 界面美观
```
