## 1. 问题摘要

### 1.1 触发原因

在项目整体评估阶段发现，现有应用的UI设计缺乏统一的设计系统，导致用户界面体验不一致。

### 1.2 核心问题

| 问题类别 | 具体表现 | 严重程度 |
|----------|----------|----------|
| 颜色系统 | 有完整颜色系统(colors.ts)但页面大量使用硬编码值 | 高 |
| 字体系统 | 有完整字体系统(typography.ts)但页面硬编码字体大小 | 高 |
| 布局系统 | 有响应式系统(tablet.ts)但页面使用魔法数字 | 中 |
| 图标设计 | 使用 emoji 替代专业图标 | 中 |
| 样式重复 | 大量重复样式代码，缺乏组件化 | 中 |
| 间距不一致 | padding/margin 值混乱(8,12,15,16,20,24,40,60等) | 中 |

### 1.3 证据

**HomeScreen.tsx 示例:**
- 硬编码颜色: `#007bff`, `#2196F3`, `#4CAF50`, `#f5f5f5`
- 硬编码字体: 24, 16, 17, 13, 14, 18
- Emoji图标: 📚, 📊, 💡, 📝

**ProfileScreen.tsx 示例:**
- 硬编码颜色: `#f5f5f5`, `#fff`, `#666`, `#999`, `#333`
- 硬编码字体: 32, 24, 14, 16

### 4.4 设计系统规范

**间距系统 (spacing.ts):**
| 名称 | 值 | 用途 |
|------|-----|------|
| xs | 4dp | 最小间距 |
| sm | 8dp | 紧凑间距 |
| md | 12dp | 标准间距 |
| lg | 16dp | 宽松间距 |
| xl | 20dp | 大间距 |
| xxl | 24dp | 区块间距 |
| xxxl | 32dp | 大区块间距 |

**圆角系统 (borderRadius.ts):**
| 名称 | 值 | 用途 |
|------|-----|------|
| sm | 4dp | 小圆角 |
| md | 8dp | 中圆角 |
| lg | 12dp | 大圆角 |
| xl | 16dp | 卡片圆角 |
| full | 9999dp | 完全圆形 |

**图标规范:**
- 使用 MaterialIcons
- 禁止使用 emoji 替代功能图标
- 统一图标尺寸: 16/20/24/28/32/48dp

**字体规范:**
- 使用系统默认字体
- 保持正常字号，无需为平板放大
