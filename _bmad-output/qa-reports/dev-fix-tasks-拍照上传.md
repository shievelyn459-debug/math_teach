**影响**:
- E2E测试无法正常调用OCR服务
- 4个边界条件测试失败

**错误信息**:
```

```

**失败位置**:
- `src/services/ocrService.ts` - OCRService类
- `recognizeQuestionType` 静态方法

**根本原因**:
OCRService是一个单例类，`recognizeQuestionType`是静态方法，但在测试环境中可能没有被正确初始化或导出。

**需要Dev确认**:
1. OCRService是如何导出的？（类还是实例）
2. `recognizeQuestionType`是静态方法吗？
3. 是否需要调整测试调用方式？

**预期修复方式**:
选项A: 如果是静态方法，测试应改为：
```typescript
import { OCRService } from '../../services/ocrService';

// 调用静态方法
await OCRService.recognizeQuestionType(text);
```

选项B: 如果需要实例，修改OCRService导出：
```typescript
// 确保单例正确导出
export const ocrService = OCRService.getInstance();
```
