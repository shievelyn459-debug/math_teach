## 1. 设计背景

### 1.1 当前问题

| 问题 | 影响 |
|------|------|
| 预览区域缺少引导 | 家长不知道如何构图 |
| 控制按钮无文字标签 | 需要猜测图标含义 |
| 拍照后无质量反馈 | 可能上传模糊照片 |
| 界面偏冷色调 | 增加家长焦虑感 |

### 1.2 用户画像

**主要用户：家长**

| 特征 | 描述 |
|------|------|
| 年龄 | 30-45岁 |
| 技术熟练度 | 中等偏低 |
| 使用场景 | 晚上辅导孩子作业时 |
| 主要目标 | 快速拍好题目，得到解答 |
| 痛点 | 担心照片拍不好、操作复杂 |

### 1.3 设计目标

1. **简化操作** - 3步内完成拍照上传
2. **清晰引导** - 让家长知道如何拍出好照片
3. **温暖体验** - 减少焦虑，营造轻松氛围
4. **容错设计** - 允许轻松重拍

## 8. 无障碍设计

### 8.1 文字大小

- 支持系统字体缩放
- 最小可读字号 16sp
- 重要信息使用 18sp 以上

### 8.2 颜色对比度

- 主文字对比度 ≥ 4.5:1
- 次文字对比度 ≥ 3:1
- 按钮文字对比度 ≥ 4.5:1

### 8.3 触控目标

- 按钮最小点击区域 48×48dp
- 按钮间距 ≥ 16dp
- 避免过于密集的操作区

### 8.4 辅助功能

- 所有图片提供 alt 文本
- 按钮提供 accessibilityLabel
- 支持屏幕阅读器
